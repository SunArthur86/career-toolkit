## 1. 25匹马5赛道找前3名？

> **难度**: medium | **分类**: 智力题 | **ID**: 54

#### 🎯本质
在**没有计时器**、只有5条赛道的情况下，用最少比赛次数从25匹马中找出速度**前3名**。核心思路是分组+淘汰+缩小候选范围。

#### 🧒类比
像学校选3个最快的人去参加百米赛——先按班级比，再让各班第一名比，最后只看有可能进前三的选手。

#### 📊图解
25匹马分5组(A-E), 每组5匹

第1-5次: 每组各跑1次(共5次)
  A组: A1>A2>A3>A4>A5
  B组: B1>B2>B3>B4>B5
  ...E组同理

第6次: 5个第一名跑
  假设结果: A1>B1>C1>D1>E1
  → A1是全场第1名 ✅
  → D组E组全部淘汰
  → C2,C3也淘汰(C1最多第3)

候选分析:
  第2名可能在: A2, B1
  第3名可能在: A3, B2, C1

第7次: A2, A3, B1, B2, C1跑
  取前2名 → 全场第2和第3 ✅
#### 🔧详解
关键推理：第6次比赛确定了A1第1后，D1最多只能排第4（前面有A1,B1,C1），所以D组E组全部淘汰。C1最多第3，那C2及以后淘汰。B1最多第2，B2最多第3，B3淘汰。A2最多第2，A3最多第3，A4淘汰。所以最终候选只有A2,A3,B1,B2,C1共5匹，正好一场比赛搞定。

#### 💻代码
// 模拟验证(假设每匹马有固定速度)
const horses = Array.from({length:25}, (_,i) => ({
  id: i, speed: 25-i // 简化为速度递减
}));
// 分5组
const groups = Array.from({length:5}, (_,i) =>
  horses.slice(i*5, i*5+5)
    .sort((a,b) => b.speed - a.speed)
);
// 第6次:各组第一
const firsts = groups.map(g => g[0])
  .sort((a,b) => b.speed - a.speed);
const rank1 = firsts[0]; // A1
// 第7次:候选马
const candidates = [
  groups[0][1], groups[0][2], // A2,A3
  groups[1][0], groups[1][1], // B1,B2
  groups[2][0]               // C1
].sort((a,b) => b.speed - a.speed);
const rank2 = candidates[0];
const rank3 = candidates[1];

## 2. 烧绳子计时15分钟？

> **难度**: easy | **分类**: 智力题 | **ID**: 78

#### 🎯 本质
给定两根粗细不均匀的绳子，每根烧完恰好1小时。由于不均匀，不能通过烧一半来量时间。但**同时从两端点燃**一根绳子，烧完时间一定是30分钟（因为两端同时燃烧，速度加倍）。

#### 🧒 类比
想象一条不规则的隧道——一个人从一端走到另一端要1小时（绳子烧完），但两个人从两端同时出发，一定会在半小时相遇（绳子烧完=两端火焰相遇）。

#### 📊 图解
绳子A: 60分钟烧完
绳子B: 60分钟烧完

步骤:
  t=0min:  A两端同时点 + B一端点
  t=30min: A烧完(两端点=30分钟)
           B已烧30分钟,还剩30分钟的量
           → 立即点燃B的另一端!
  t=45min: B烧完(剩余30分钟的量从两端烧=15分钟)

所以: 从A烧完到B烧完 = 15分钟 ✅
     从开始到B烧完 = 45分钟
#### 🔧 详解
核心洞察是**从两端同时燃烧，时间减半**。不管绳子多不均匀，两端火焰一定会在中间某处相遇，总时间恰好是一半。同理，一根绳子若一端已燃烧了x分钟，剩余量从两端同时燃烧需要 (60-x)/2 分钟。灵活运用这个原理可以量出各种时间段。

#### 💻 代码
// 模拟验证
class Rope {
  constructor() { this.burnTime = 60; }
  burnFromOneEnd() { return this.burnTime; }
  burnFromBothEnds() { return this.burnTime / 2; }
}
const A = new Rope();
const B = new Rope();
// A两端点: 30分钟烧完
const t1 = A.burnFromBothEnds(); // 30
// 此时B一端烧了30分钟,剩余30分钟的量
// 再点B另一端: 30/2=15分钟烧完
const t2 = 30 / 2; // 15
console.log("15分钟 =", t2, "分钟");
// 如何量45分钟?
console.log("45分钟 =", t1 + t2, "分钟");
#### ❓ 追问
如何用两根绳子量出30分钟？答：只把一根绳子两端同时点燃即可。如何量出7.5分钟？答：在15分钟的方案基础上，当B两端点燃的同时，用第三根绳子（如果有）继续操作。实际题目中绳子数量有限，只能量出15/30/45/60这几个时间段。

## 3. 8个球其中一个偏重，秤几次找到？

> **难度**: hard | **分类**: 智力题 | **ID**: 79

#### 🎯 本质
这是一道**信息量优化**问题。天平每次称量有三种结果（左重、右重、相等），因此一次称量最多区分3种情况。关键是通过**合理分组**，让每次称量都能最大程度缩小候选范围。

#### 🧒 类比
像玩猜数字游戏——聪明的人每次都把范围切成三份而不是两份，因为天平能告诉你三件事：A重、B重、或一样重。

#### 📊 图解
8个球: ①②③④⑤⑥⑦⑧

第1次称量: ①②③ vs ④⑤⑥

  情况A: 相等 → 偏重在⑦⑧中
    第2次: ⑦ vs ⑧
      重的那个就是 ✅

  情况B: 左重 → 偏重在①②③中
    第2次: ① vs ②
      左重→① ✅
      右重→② ✅
      相等→③ ✅

  情况C: 右重 → 偏重在④⑤⑥中
    第2次: ④ vs ⑤
      左重→④ ✅
      右重→⑤ ✅
      相等→⑥ ✅

结论: 只需2次 ✅
公式: 3^n >= 球数
  3^2=9 >= 8 ✅
#### 🔧 详解
核心思路是每次称量都将候选球分为三组，其中两组数量相同放上天平。天平的三种结果对应三组候选，无论哪种结果都能将候选范围缩小到约1/3。推广：n次称量最多从 (3^n-3)/2 个球中找出1个异常球（已知偏重或偏轻方向时最多3^n个）。

#### 💻 代码
// 模拟验证
function findHeavyBall(balls) {
  // balls数组: 7个重量1, 1个重量2
  const sum = arr => arr.reduce((a,b) => a + b, 0);
  // 第1次: 3 vs 3
  const g1 = balls.slice(0, 3);
  const g2 = balls.slice(3, 6);
  const g3 = balls.slice(6, 8);
  const s1 = sum(g1), s2 = sum(g2);
  if (s1 === s2) {
    // 在g3(2个)中
    return g3[0] > g3[1] ? 6 : 7; // 索引
  } else if (s1 > s2) {
    // 在g1(3个)中
    return g1[0] > g1[1] ? 0
         : g1[1] > g1[0] ? 1
         : 2;
  } else {
    // 在g2(3个)中
    return g2[0] > g2[1] ? 3
         : g2[1] > g2[0] ? 4
         : 5;
  }
}

#### 🧒 类比

  ...

## 9. 过桥问题？

#### 🧒 类比
快的当跑腿——最快的回来送手电筒。但慢的一起走省时间。

#### 📊 图解
策略:
1.1+2过(2分)
2.1回(1分)
3.5+10过(10分)
4.2回(2分)
5.1+2过(2分)
总计: 17分
#### 🔧 详解
关键：让最慢的两人一起过桥(不要分别和快的人过)。方案：快的先过+快的回来→慢的一起过→快的回来接。

#### 💻 代码
function bridgeTime(times){
  times.sort((a,b)=>a-b);
  let total=0;
  while(times.length>3){
    const [a,b,...rest]=times;
    // 策略: a+b+a + c+d+b = 2b+a+d
    total+=2*b+a+times[times.length-1];
    times.pop();times.pop();
  }
  if(times.length===3) total+=times[0]+times[1]+times[2];
  else if(times.length===2) total+=times[1];
  else total+=times[0];
  return total;
}
#### ❓ 追问

## 12. 海盗分金币？

> **难度**: hard | **分类**: 智力题 | **ID**: 458

#### 🎯 本质
5海盗A/B/C/D/E分100金币。A提方案>半数同意则通过否则A死B提。A说**97:0:1:2:0**或**97:0:1:0:2**。

#### 🧒 类比
从后往前推——假设只剩最后两人时的分配，再往前推导每个人的最优策略。

#### 📊 图解
倒推:
只剩E: E拿100
剩D+E: D需D同意→D:100,E:0
剩C+D+E: C需1人支持→C:99,D:0,E:1
剩B+C+D+E: B需1人→B:99,C:0,D:1,E:0
A+B+C+D+E: A需2人→A:97,B:0,C:1,D:2,E:0
#### 🔧 详解
从最少人开始倒推。只剩E时E拿全部。剩D和E时D自己同意就够了(D:100,E:0)。剩CDE时C需要一个人支持给E 1个(E原来0现在1会同意)。如此倒推。A需要2人支持，给C 1个(原来0)、D 2个(原来1)就够了。
