## 3. 什么是死锁？如何避免？

> **难度**: easy | **分类**: 操作系统 | **ID**: 75

#### 🎯本质
死锁是**两个或多个进程互相等待对方释放资源**，导致所有进程都无法继续执行的僵局。产生死锁必须同时满足四个必要条件。

#### 🧒类比
两个人过独木桥，走到中间各不相让——甲等乙退，乙等甲退，谁也不动，这就是死锁。

#### 📊图解
死锁四个必要条件(缺一不可):
① 互斥条件: 资源同一时间只能被一个进程使用
② 持有并等待: 持有资源的同时等待其他资源
③ 不可抢占: 已获得的资源不能被强制夺走
④ 循环等待: 进程间形成环形等待链
   P1→等P2的资源→P2等P3的资源→P3等P1

避免策略(破坏四个条件):
✓ 破坏互斥: 允许共享(如只读文件)
✓ 破坏持有并等待: 一次性申请所有资源
✓ 破坏不可抢占: 超时后释放已持有资源
✓ 破坏循环等待: 资源有序分配(编号顺序)

检测与恢复:
  银行家算法: 分配前检查是否安全
  超时机制: 等待超时自动回退
  图算法: 检测等待图中的环
#### 🔧详解
**最实用的避免方法是资源有序分配**：给所有资源编号，进程必须按编号递增顺序申请资源，这样就不会形成环形等待。数据库中常见死锁：两个事务分别持有对方需要的行锁。数据库会自动检测并回滚其中一个事务。

## 4. 虚拟内存是什么？

> **难度**: medium | **分类**: 操作系统 | **ID**: 76

#### 🎯本质
虚拟内存是一种**内存管理技术**，让每个进程以为自己拥有完整的、连续的内存空间，实际通过**页表映射**到物理内存和磁盘swap区，实现内存的按需分配和隔离。

#### 🧒类比
虚拟内存像一个共享办公空间——每个创业者（进程）以为自己有一整栋楼，实际是按需分配工位（物理内存），不常用的资料放到仓库（磁盘），需要时再取回来。

#### 📊图解
虚拟地址空间(每个进程独立):
┌──────────────────────┐ 0xFFFFFFFF
│   内核空间           │ (所有进程共享)
├──────────────────────┘ 0xC0000000
│   栈(stack) ↓增长    │
│   ...                │
│   共享库             │
│   ...                │
│   堆(heap) ↑增长     │
├──────────────────────┤
│   BSS段(未初始化)    │
│   数据段(已初始化)   │
│   代码段(text)       │
└──────────────────────┘ 0x00000000

地址转换过程:
虚拟地址 → 页表查找 → 物理地址
  ↓ 如果页不在内存中
  触发缺页中断(page fault)
  → 从磁盘/swap加载到物理内存
  → 更新页表
  → 重新执行指令

TLB(Translation Lookaside Buffer):
  页表的高速缓存
  命中TLB → 1个时钟周期
  未命中 → 需要查页表(内存访问)

页面置换算法:
  LRU(最近最少使用) - 最常用
  FIFO(先进先出)
  Clock(时钟算法)
#### 🔧详解
虚拟内存的核心好处：**进程隔离**（每个进程独立地址空间，互不干扰）、**内存保护**（防止越界访问）、**按需分配**（不用一次加载全部）。Linux默认页大小4KB。swap空间在物理内存不足时充当扩展，但swap频繁会导致系统变慢（应通过swappiness参数调优）。

#### 💻代码

free -h          # 查看内存和swap
vmstat 1         # 每秒查看虚拟内存统计
cat /proc/meminfo # 详细内存信息

# 调整swap倾向(0-100,值越少越少用swap)
sudo sysctl vm.swappiness=10

# 查看进程虚拟内存映射
cat /proc/PID/maps
pmap -x PID      # 进程内存映射详情

  int stack = 1;  // 栈
  int *heap = malloc(sizeof(int)); // 堆
  printf("代码段: %p
", (void*)main);
  printf("数据段: %p
", (void*)&global);

#### ❓追问
什么是缺页中断？答：CPU访问的虚拟页不在物理内存中，触发中断，OS从磁盘加载该页到内存，更新页表后重新执行指令。什么是内存映射文件mmap？答：将文件映射到虚拟地址空间，像访问内存一样访问文件，避免read/write系统调用。

## 5. 用户态和内核态？

> **难度**: medium | **分类**: 操作系统 | **ID**: 77

#### 🎯 本质
操作系统通过**特权级**将运行环境分为用户态（Ring 3）和内核态（Ring 0）。用户态下程序只能访问受限资源，内核态拥有对硬件和内存的完全控制权。两者之间的切换是操作系统安全与性能平衡的核心机制。

#### 🧒 类比
用户态像普通员工在大厅办公（只能用公共设备），内核态像进入机房（有全部权限）。要进机房必须走专用通道（系统调用），门卫会检查你的请求是否合法。

#### 📊 图解
用户态(Ring 3)       内核态(Ring 0)
  应用程序              内核
  浏览器/Node          文件系统/网络
  受限指令              硬件操作
  ↓ 系统调用/中断
  保存上下文→切换栈→提权
  → 执行内核代码
  → 恢复上下文→降权
  → 返回用户态

切换触发:
  ① 系统调用(read/write/ioctl)
  ② 中断(时钟/键盘/网络)
  ③ 异常(缺页/除零)

每次切换开销:
  保存/恢复寄存器+切换栈
  约200-1000个时钟周期
#### 🔧 详解
用户态通过**系统调用（syscall）**请求内核服务，CPU从Ring 3切换到Ring 0，内核执行完后返回结果。**中断**是硬件主动通知CPU的事件（如网卡收到数据），也会触发切换。**缺页中断**是访问未映射的虚拟地址时触发的异常。频繁的用户态/内核态切换是性能瓶颈之一，如高频IO操作可考虑用mmap减少切换。

#### 💻 代码
// Node.js中的用户态/内核态体现
const fs = require("fs");
// readFileSync: 用户态→内核态→等磁盘→返回
const data = fs.readFileSync("big.txt");
// 整个过程阻塞在内核态等待

// 异步版本:更快但同样有切换
fs.readFile("big.txt", (err, data) => {
  // 内核读完通过事件通知回来
});

// mmap减少切换(零拷贝)
// 普通read: 磁盘→内核缓冲→用户缓冲 (2次拷贝)
// mmap: 磁盘→内核缓冲,用户空间直接映射 (0次额外拷贝)

// 查看Node进程的系统调用
// strace -c node app.js
// 可以看到read/write/mmap等调用次数和时间
#### ❓ 追问
为什么不让所有程序都在内核态运行？答：安全性和稳定性——恶意或有bug的程序可能直接操作硬件导致系统崩溃，用户态隔离保护了内核。什么是零拷贝技术？答：sendfile/mmap等让数据不经过用户空间直接在内核完成传输，减少CPU拷贝和上下文切换。

#### 📊 图解
进程: 独立地址空间/文件描述符/堆栈
线程: 共享进程资源/独立栈和寄存器

## 7. 死锁？

> **难度**: medium | **分类**: 操作系统 | **ID**: 312

#### 🎯 本质
死锁是**多个进程互相等待**对方持有的资源导致都无法继续。四个必要条件：互斥/持有并等待/不可剥夺/循环等待。

#### 🧒 类比
死锁像两个人过独木桥——都走到中间不肯退，谁也过不去。

#### 📊 图解
死锁四条件(同时满足):
1.互斥: 资源同一时刻只能被一个进程使用
2.持有并等待: 持有资源同时等待其他
3.不可剥夺: 不能强行夺走资源
4.循环等待: 形成等待环路

预防: 破坏任一条件
#### 🔧 详解
只要破坏四个条件之一就能预防死锁。按顺序申请资源(破坏循环等待)。银行家算法避免死锁(检查分配是否安全)。超时放弃(破坏不可剥夺)。检测死锁用资源分配图。数据库的死锁检测会自动回滚代价最小的事务。

#### 💻 代码
// 避免死锁: 按顺序加锁
// ❌ 可能死锁
lock(A); lock(B);  // 线程1
lock(B); lock(A);  // 线程2
// ✅ 统一顺序
lock(A); lock(B);  // 所有线程都先A后B
#### ❓ 追问
什么是活锁？答：进程不断改变状态但无法前进(像两个人互相让路)。银行家算法？答：分配资源前检查是否导致不安全状态。

#### 🎯 本质
虚拟内存让每个进程以为自己有**独立的连续内存空间**，实际物理内存通过页表映射。

#### 🧒 类比
虚拟内存像图书馆的预约系统——每个人以为书架上都是自己的书，实际是按需从仓库调。

虚拟地址 → [页表] → 物理地址

分页:
  页(Page): 虚拟内存固定大小块(4KB)
  页框(Frame): 物理内存块
  页表: 虚拟页→物理框映射

TLB: 页表缓存(加速地址翻译)
#### 🔧 详解
每个进程有自己的虚拟地址空间(独立互不影响)。MMU(内存管理单元)将虚拟地址翻译为物理地址。页表存储映射关系。TLB是页表的缓存加速翻译。缺页中断(Page Fault)时从磁盘加载页面。页面置换算法：LRU/Clock/FIFO。

// 查看进程内存

// 虚拟内存统计

#### ❓ 追问
什么是缺页中断？答：访问的虚拟页不在物理内存中触发中断从磁盘加载。页面置换算法LRU的原理？答：替换最久未使用的页面。

## 9. 用户态和内核态？

> **难度**: easy | **分类**: 操作系统 | **ID**: 314

#### 🎯 本质
CPU有**两种运行级别**：用户态(受限)和内核态(完全权限)。系统调用从用户态切换到内核态。

#### 🧒 类比
用户态像普通员工(权限有限)，内核态像CEO(完全控制)。需要特殊操作要走审批(系统调用)。

#### 📊 图解
用户态: 应用程序运行(受限访问)
内核态: 操作系统内核运行(完全权限)

切换方式:
1.系统调用(主动)
2.中断(外部)
3.异常(错误)
#### 🔧 详解
应用程序在用户态运行不能直接操作硬件。需要操作硬件(文件/网络/内存)时通过系统调用切换到内核态。系统调用有开销(上下文切换)。中断和异常也会触发切换。Linux有约400个系统调用。

#### 💻 代码

系统调用的开销？答：上下文切换+权限检查+可能的缓存失效。

## 10. 进程调度算法？

> **难度**: medium | **分类**: 操作系统 | **ID**: 315

#### 🎯 本质
进程调度决定**CPU分配给哪个进程**。常见算法：FCFS/SJF/优先级/时间片轮转/多级反馈队列。

#### 🧒 类比
调度像食堂排队——先来先服务(FCFS)、快餐优先(SJF)、VIP优先(优先级)、每人限时(时间片)。

#### 📊 图解
调度算法:
FCFS: 先来先服务(简单但不公平)
SJF: 最短作业优先(平均等待短)
优先级: 按优先级(可能饥饿)
时间片轮转(RR): 每人固定时间
多级反馈队列: 动态调整优先级
#### 🔧 详解
FCFS简单但短作业可能等很久(护航效应)。SJF理论最优但不知道作业长度。时间片轮转公平但切换开销。多级反馈队列综合最优(短作业优先+长作业不饥饿)。Linux用CFS(完全公平调度器)。

#### 💻 代码
// Linux查看调度策略
chrt -p 
// 设置优先级
renice -n 10 -p 
// nice值: -20(最高)到19(最低)
#### ❓ 追问
什么是进程饥饿？答：低优先级进程永远得不到CPU。CFS原理？答：虚拟运行时间最小的进程优先运行。

#### 🎯 本质
文件系统负责**组织/存储/检索**磁盘上的文件。常见：ext4/NTFS/APFS/HDFS。

#### 🧒 类比
文件系统像图书馆分类系统——按规则组织书籍(文件)便于查找。

#### 📊 图解
常见文件系统:
Linux: ext4(日志)/XFS(大文件)/Btrfs
Windows: NTFS/exFAT/FAT32
macOS: APFS
分布式: HDFS/Ceph/GlusterFS

概念:
inode/目录/日志/权限
#### 🔧 详解
ext4是Linux默认文件系统(支持日志恢复)。inode存储文件元数据(大小/权限/时间戳)。目录是文件名到inode的映射。日志(journal)记录操作用于崩溃恢复。权限:rwx(读/写/执行)对用户/组/其他。

#### 💻 代码

#### ❓ 追问
什么是inode？答：存储文件元数据(除文件名外)的数据结构。软链接和硬链接的区别？答：软链接存路径(可跨文件系统)，硬链接存inode引用(同文件系统)。

## 12. 什么是中断？

> **难度**: easy | **分类**: 操作系统 | **ID**: 317

#### 🎯 本质
中断是**CPU暂停当前任务处理紧急事件**的机制。硬件中断(外部设备)和软件中断(系统调用/异常)。

#### 🧒 类比
中断像上课时电话响了——暂停讲课(当前任务)去接电话(处理中断)然后回来继续。

#### 📊 图解
中断类型:
硬件中断: 时钟/键盘/网卡/磁盘
软件中断: 系统调用/除零/缺页

处理流程:
1.保存当前上下文
2.查找中断向量表
3.执行中断处理程序
4.恢复上下文继续执行
#### 🔧 详解
硬件中断由外部设备触发(异步)。软件中断由程序触发(系统调用int 0x80或syscall)。中断处理要快(不能长时间阻塞)。上半部(紧急处理)/下半部(延迟处理)机制。中断描述符表(IDT)存储处理程序地址。

#### 💻 代码
// Linux查看中断
cat /proc/interrupts
watch -n1 cat /proc/interrupts
// 禁用中断(内核编程)
local_irq_disable();
// critical section
local_irq_enable();
#### ❓ 追问
什么是中断上下半部？答：上半部快速处理紧急部分，下半部稍后处理耗时部分。

## 13. 并发和并行？

> **难度**: medium | **分类**: 操作系统 | **ID**: 318

#### 🎯 本质
**并发**是多个任务交替执行(单核)。**并行**是多个任务同时执行(多核)。

#### 🧒 类比
并发像一个人交替做几件事(炒菜+切菜轮着来)，并行像多个人同时做(两个人同时炒菜)。

#### 📊 图解
并发: 交替执行(单核CPU时间片)
  → 看起来同时(实际切换)
并行: 真正同时(多核CPU)
  → 物理上同时执行

相关:
同步/异步
阻塞/非阻塞
#### 🔧 详解
并发是逻辑上的同时(任务管理)。并行是物理上的同时(多核执行)。单核CPU只能并发不能并行。多核CPU可以并发也可以并行。异步I/O+事件循环是并发的一种实现(Node.js)。Go的goroutine是并发的轻量实现。

#### 💻 代码
// Node.js 并发(异步I/O)
const [users,posts]=await Promise.all([
  fetchUsers(),  // 并发执行
  fetchPosts()
]);
// Worker 并行
const workers=[new Worker('./w1.js'),new Worker('./w2.js')];
#### ❓ 追问
什么是协程？答：用户态的轻量线程，由程序控制切换(Go goroutine/JS async)。

## 14. I/O多路复用？

> **难度**: medium | **分类**: 操作系统 | **ID**: 319

#### 🎯 本质
I/O多路复用让**单个线程同时监听多个I/O**。select/poll/epoll是Linux三种实现。

#### 🧒 类比
I/O多路复用像一个前台同时接多部电话——有电话响(就绪)才去接而不是死等一部。

#### 📊 图解
实现方式:
select: 遍历所有fd(慢,上限1024)
poll: 类似select无上限
epoll: 事件驱动(快,无上限)
  - epoll_create/epoll_ctl/epoll_wait

macOS: kqueue
Windows: IOCP
#### 🔧 详解
select/poll每次调用都要传入所有fd并遍历(性能差)。epoll只在fd就绪时通知(事件驱动性能好)。epoll的LT(水平触发)和ET(边缘触发)。Node.js和Nginx底层用epoll/kqueue。这就是单线程能处理高并发的秘密。

#### 💻 代码
// Node.js底层使用epoll
// 等价伪代码
const epfd=epoll_create();
epoll_ctl(epfd,ADD,socketfd,EPOLLIN);
while(true){
  const events=epoll_wait(epfd,maxEvents,timeout);
  events.forEach(e=>handleEvent(e));
}
#### ❓ 追问
LT和ET的区别？答：LT只要还有数据就持续通知，ET只在状态变化时通知一次。

## 15. Linux常用命令？

> **难度**: easy | **分类**: 操作系统 | **ID**: 320

#### 🎯 本质
Linux常用命令：**文件操作(ls/cp/mv/rm)、文本处理(grep/sed/awk)、进程管理(ps/top/kill)、网络(netstat/curl)**。

#### 🧒 类比
Linux命令像瑞士军刀——每把刀(命令)一个用途，组合使用威力无穷。

#### 📊 图解
文件: ls/cd/pwd/mkdir/rm/cp/mv/chmod
文本: cat/head/tail/grep/find/wc/sort/uniq
进程: ps/top/htop/kill/nice
网络: netstat/ss/curl/wget/ping/ifconfig
系统: df/du/free/uptime/dmesg
#### 🔧 详解
ls列出目录，grep搜索文本，find查找文件，ps查看进程，netstat查看网络连接，tail -f实时查看日志。管道(|)连接命令。重定向(>/>>/>/)输出到文件。top/htop实时监控资源。

#### 💻 代码
// 实用组合
# 查找大文件
find / -size +100M -type f
# 统计代码行数
find . -name '*.js' | xargs wc -l
# 实时日志
tail -f app.log | grep ERROR
# 端口占用
lsof -i :8080
# 批量杀进程
ps aux | grep node | awk '{print $2}' | xargs kill
#### ❓ 追问
什么是管道(|)？答：将前一个命令的输出作为后一个命令的输入。
